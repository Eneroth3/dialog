SketchUp callbacks are assigned to elements with the class dlg-callback-{action} where {action} is the callback name defined in Ruby.

If the callback method is missing a warning is logged.

If you first want to process data on the JS side before making a callback, you can define your own onclick function or event listener instead of setting the class.

An access key can be specified to the data-access-key attribute. The user can quickly trigger/focus a command by pressing Alt+{access key}. This can speed up the work flow for the user, especially when they used a keyboard shortcut, rather than a mouse click, to open the dialog in the first place. Microsoft developer guidelines for access keys.







If a label doesn't contain the specified access key a warning is logged.


Some commands are automatically assigned shortcuts. The following keys attempts to activate element with classes in the order given below:

  • Enter - dlg-default-action, dlg-callback-yes, dlg-callback-ok, dlg-callback-close
  • Esc - dlg-callback-cancel, dlg-callback-no, dlg-callback-ok, dlg-callback-close
  • F1 - dlg-callback-help

NOTE: If the element dlg-callback-cancel is missing, Esc will try activating commands that may represent a positive response. If there is a cancel button, make sure to give it the correct class, or Esc may lead to an action being carried out.

In addition to defining shortcuts, the dlg-default-action class makes the command focused on page load. Use this class for default buttons, such as OK or Yes.
